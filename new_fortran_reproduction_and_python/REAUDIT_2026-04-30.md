## 4. Issues that remain

### 4.1 Fig 2 L convention is now resolved

The paper's Fig 2 panels have axes labelled 0 and 9, so the plotted
grid is a **10×10 site grid**.  The corrected Fortran Fig 2 drivers use
`integer, parameter :: L = 10` with sites `0..L-1`, which therefore
matches the displayed Fig 2 lattice.

Authors' `TRW.py` uses a different variable convention: its `L_A = 9`
also represents sites `0..9`, i.e. the same 10×10 physical grid.  The
correct Fig 2 cross-check is therefore

```text
Fortran L_F = 10  <->  authors' TRW.py L_A = 9
```

Do **not** bump the Fig 2 Fortran drivers to `L = 11` for the present
Fig 2 reproduction; that would switch to an 11×11 grid and no longer
match the axes in the paper panel.

### 4.2 Stale Fig 3 convention comments

Lines 18–28 say:

```
!   - Lattice IS the L × L playground.  Walker occupies any site in
!     [0, L-1] × [0, L-1].  No surrounding wall ring.
!   - Edge sites: x == 0 .or. x == L-1 .or. y == 0 .or. y == L-1
!     → n_edge = 4L − 4
!   - Bulk sites: 1 ≤ x ≤ L-2 .and. 1 ≤ x ≤ L-2
!     → n_bulk = (L − 2)²
```

But the actual code does

```fortran
L_paper = L_list(iL)             ! 4, 9, 19, 49 — paper labels
L_cur   = L_paper + 1            ! authors' convention: L=N ⇒ (N+1) sites
n_edge  = 4 * L_cur - 4
n_bulk  = (L_cur - 2) ** 2
```

So the Fig 3 drivers use the authors' convention: the legend label is
`L_paper`, while the simulated site count is `L_cur = L_paper + 1`.
No physics impact — every written summary row tags the row with
`L_paper`, and the cross-check scripts build the OBC matrix with
`L_A = L_paper`, which matches.  The comments in
`tcrw_fig3a.f90`, `tcrw_fig3b.f90`, `tcrw_fig3f.f90`, and
`tcrw_fig3g.f90` have now been updated to say this explicitly.

### 4.3 Cross-check script paths are now local

The machine-specific fallback paths have been removed from the active
cross-check helpers.  Scripts now resolve their dependencies relative to
their own folder:

```python
HERE = os.path.dirname(os.path.abspath(__file__))
ROOT = os.path.dirname(HERE)
TRW_PATH = os.path.join(HERE, "TRW._original_code_by_paperauthors.py")
```

This keeps the scripts portable across the new workspace path and avoids
stale references to older sandbox or assistant folders.
